> **출처:** 아카이브(네이버 카페 굿머닝, clubid=29082876) 2018~2026 스승님 글을 RAG 검색으로 추출.
> **주의:** 이건 한 스승님의 **재량적 코멘트**를 의미검색으로 뽑은 것(질의별 top-6). 완전 추출 아님.
> 각 규칙은 "스승님 주장"이지 검증된 사실이 아니다 → **백테스트로 수익성 검증이 전제.**

## AI 코딩 프롬프트 (아래를 그대로 코딩 AI에게 주면 됨)

```markdown
# 역할
너는 한국 주식 백테스트 엔진을 만드는 퀀트 개발자다. 아래는 한 트레이딩 멘토의 재량적 규칙을 정량화한 사양이다.
각 규칙을 config로 토글·파라미터화 가능하게 구현하고, 규칙 조합의 성과(특히 MDD)를 측정하라.
멘토 주장은 검증된 사실이 아니므로, 목적은 "이 규칙들이 실제로 수익/MDD 개선을 주는가"의 검증이다.

# 데이터 입력 (가정)
- 필수: 종목별 일봉 OHLCV, 지수(KOSPI·KOSDAQ) 일봉.
- 선택(있으면 정확도↑): 분봉(1·5분), 시장폭(상승/하락 종목수), 동시호가/호가잔량.
- 각 규칙에 data_granularity 표기: [일봉가능] / [분봉필요](없으면 자동 비활성 + 경고).

# 파라미터 표기
- 〔M〕 = 멘토 명시값(원문 근거). 〔A〕 = 가정값(백테스트로 튜닝, 기본값 제시).

## R1. 시장 레짐 스위치  [일봉가능, 시장폭 있으면 정확]  (#7938, #129709)
- VOLATILE: 지수_당일수익률 < 〔A -1.0%〕 AND 하락종목비율 > 〔A 70%〕(시장폭 없으면 지수만)
- BOX: 최근 〔A 20〕일 지수 고저폭 < 〔A 4%〕 (또는 ADX<20)
- 액션: VOLATILE → 오버나잇 ≤20%〔M〕, 신규는 당일청산(단타)만〔M〕 / BOX → 시초가·동시호가 추격금지〔M〕, 진입은 장중 눌림에서만〔M〕

## R2. 장대양봉/상한가 청산  [일봉가능] ★  (#40599)
- 트리거: 보유종목 당일 일간수익률 ≥ 〔M +10%〕(장대양봉) 또는 상한가(+30%).
- 보수형: 장대양봉 당일 종가에 30% 분할매도, 70% 익일 보유〔M〕.
- 하드청산1: 익일 하락 ≥ 〔M -10%〕 → 전량매도.
- 하드청산2(일반화): 가격이 "허리" 이탈 → 전량매도. 허리 = 장대양봉일 상승분의 30% 되돌림 지점〔M〕.
  (되돌림 기준선=고-저 / 시-종 / 전일종가 중 무엇인지 옵션화해 비교)

## R3. 손절 분기  [일봉가능]  (#91752, 보조 #43209 #89531)
- MOMENTUM(세력·수급주, R6 진입) → 손절 〔M -3~4%, 기본 -3.5%〕
- FUNDAMENTAL(실적·성장 보유) → 손절 〔M -10%〕
- (옵션) BEGINNER 전체 -5% 캡〔M〕. 손절은 무조건 집행. 진입규칙이 분류 태그 출력.

## R4. 익절 30% 분할 + 첫 익절 후 음봉 재매수  [일봉가능]  (#26566, #53122)
- 수익 〔A +10%, 보조 #49652 "+20%서 긴장"〕 도달 시 30% 매도, 추가 상승 시 또 30%〔M〕.
- 1차 익절 후 다음봉이 음봉이면 익절한 30%를 저점 부근 재매수〔M〕.

## R5. 분할매수  [일봉가능]  (#53601, #24948)
- 선발대(초도) ≤ 〔M 20%〕 목표비중, 최소 2회 이상 분할〔M〕, 음봉 하락종목일수록 더 잘게〔M〕.
- 금지: 같은 날 등락마다 풀매수, "가격 내릴 이유 사라지기 전" 무한 물타기, 한두 호가 매집〔M〕.
  구현: 추가매수는 (시간간격 ≥ 〔A 1일〕) AND (진입논리 R6 유효) 일 때만.

## R6. 진입 셋업(봉패턴)  [A·D 분봉필요 / B·C 일봉가능]  (#67253 #78948 #80367 #89519)
- 표준 매수 셋업(#89519, 기본): 급등→급락→저점확인→반등→작은 음봉 1~2개 자리 = 분할매수(R5) 시작 → BUY.
- Setup A(아침 급락→양봉전환)[분봉]: 장초 급락 후 양봉반전=세력 개미털기 → 매수. 역패턴(아침 급등→음봉전환)=분배 → 회피.
- Setup B(눌림의 끝, #78948)[일봉]: 급등 후 눌림=작은음봉 1~2개, 그 끝에서 진입.
- Setup D(강세출발, #80367)[분봉]: 시초 강세는 추격금지, 오후 눌림 대기 후 분할매수.
- 모든 BUY는 R1 레짐 필터 통과, 진입 시 R3 분류 태그 부여.

# 포지션·자금관리
- 종목당 목표비중 〔A 자본 10%〕, 동시보유 〔A 5〕. 레짐별 총노출 캡(VOLATILE 오버나잇 ≤20%).
- 슬리피지·수수료·세금(KR 매도세 등) 파라미터화.

# 백테스트 출력
- 지표: CAGR, MDD(최우선), Sharpe, 승률, 손익비, 평균보유기간, 규칙별 기여도.
- 비교: (a) 전부 OFF(벤치마크) (b) 각 규칙 단독 ON (c) 전체 ON → MDD·CAGR 표.
- 거래로그(진입/청산 사유=발화 규칙) 필수.

# 구현 지침
- config 한 파일로 모든 〔A〕〔M〕 파라미터 노출, 규칙별 enabled 토글.
- 분봉 필요 규칙은 분봉 없으면 자동 비활성 + 경고 로그(무성 스킵 금지).
- 클래스: Regime, SignalRule(R1..R6), PositionManager, Backtester.

# 코딩 전 반드시 사용자에게 확인할 모호점
1. R2 "허리" 되돌림 기준선(고-저 / 시-종 / 전일종가).
2. R4 1차 익절 트리거 수익률(tp_trigger) 절대%.
3. R6 Setup A·D용 분봉 데이터 가용 여부.
4. R3 종목 분류(MOMENTUM vs FUNDAMENTAL) 판정 방법.
5. "급등·급락"의 정량 정의(예: 급등 = N일 +X%).
```
